# Task: fix-from

**Status**: complete
**Branch**: hatchery/fix-from
**Created**: 2026-03-27 15:45

## Objective

Fix `FileNotFoundError` when running `hatchery new <name>` in a repo with no pre-existing `.hatchery/` directory.

## Context

Commit 60c0e9d correctly moved dockerfile creation from the repo root into the worktree so it lands on the task branch. However, the worktree's `.hatchery/` subdirectory is never created before the write is attempted.

The execution order in `cmd_new()` was:
1. `tasks.ensure_tasks_dir(repo)` — creates `.hatchery/` in the **repo root** ✓
2. `git.create_worktree(...)` — creates the worktree directory ✓
3. `docker.ensure_dockerfile(worktree, ...)` — tried to write `worktree/.hatchery/Dockerfile.agent` ✗ parent missing
4. `tasks.write_task_file(worktree, ...)` — would have created `worktree/.hatchery/tasks/` (too late)

## Summary

Added `df.parent.mkdir(parents=True, exist_ok=True)` in `ensure_dockerfile()` and `config_file.parent.mkdir(parents=True, exist_ok=True)` in `ensure_docker_config()`, both in `src/seekr_hatchery/docker.py`. This is a no-op when the directory already exists, and fixes the crash for fresh repos. The fix is minimal and self-contained — no changes to call sites.
