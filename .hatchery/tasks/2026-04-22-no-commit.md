# Task: no-commit

**Status**: complete
**Branch**: hatchery/no-commit
**Created**: 2026-04-22 15:57

## Objective

Sometimes users don't want hatchery to commit any files for them (no task file, docker, etc).

We should support that

## Summary

Added a `--no-commit` flag to the `new`, `chat`, and `sandbox` commands. When set, hatchery skips all automatic git commits it would otherwise make on the user's behalf.

**Key decisions:**

- `--no-commit` implies `--no-commit-docker` behavior: Docker files are still written to the repo root (so the docker daemon can find them) but are never staged or committed.
- `no_commit` is persisted in task metadata (`~/.hatchery/tasks/<repo-id>/<name>/meta.json`) alongside the existing `no_worktree` flag, so that the `done` and `archive` commands can skip their checkpoint commit prompts without requiring the user to re-specify the flag.
- The checkpoint commit prompts in `_do_mark_done` and `cmd_archive` are skipped silently (no prompt shown) when `no_commit=True`, matching the spirit of "don't touch git for me."

**Files changed:**

- `src/seekr_hatchery/cli.py` — only file modified

**Gotchas:**

- The agent itself (Claude Code) still makes its own commits as instructed by the system prompt. `--no-commit` only suppresses *hatchery's* scaffolding commits, not agent commits.
- Existing tasks created without `--no-commit` default to `no_commit=False` via `meta.get("no_commit", False)`, so there is no migration needed.
